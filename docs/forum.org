* Researching milestones in Perseus Corpora
The TEI-encoded texts in the Perseus Corpora use many types of <div>
element and many kinds of <milestone unit="someUnit"> elements to
represent structure, as well as various types of page-break markers
<pb>. We want to analyze the use of these structuring elements across
the entire corpus in order to design strategies for chunking the TEI
texts into sequential HTML pages that are identifiable via CTS urns.

JSON shape adopted:
#+begin_src json
{
  "filename": "<stem>",
  "milestones": [{"unit": "<unit-attr>", "frequency": N}],
  "divs": [{"div_type": "<type>", "subtypes": [{"subtype": "<subtype>|null", "frequency": N}]}]

The forum spec used ={"milestone": ..., "frequency": ...}= but the key name
was ambiguous (the value is the =@unit= attribute). Normalized to =unit= for
clarity.

*** Components
- *analyze_milestones(tei_doc:Path, target_dir:Path) -> None:*

  A function that traverses a TEI document, using lxml, and compiles
  several lists:

  - a list of milestone types : organized by unit attribute.  Record the
    frequency of each type.

  - a list of div types, organized by type and subtype attributes.
    Record the frequency of each type and subtype.

   The output of this function is a JSON object:
   { "filename" : name ,
     "milestones" : [{"milestone" : milestone, "frequency" :
   frequency}],
     "divs" : [{"div_type" : [{ "subtype" : subtype "frequency"}] }]
     }

  The object should be serialized to target_dir/filename.json.

- *analyze-milestones.py --source_dir --output_dir*
  A script that traverses source_dir, applying =analyze_milestones()=
  to each TEI document (be sure to skip __cts__.xml files).
